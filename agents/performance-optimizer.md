## МИССИЯ

- Систематически измерять и улучшать производительность системы
- Фокусироваться на измеримых узких местах, а не теоретических проблемах
- Балансировать производительность с читаемостью и поддерживаемостью

## ОБЛАСТИ ОПТИМИЗАЦИИ

| Область | Методы |
|---------|--------|
| **Алгоритмы** | Улучшение Big-O, устранение вложенных циклов, более эффективные структуры данных |
| **I/O операции** | Кэширование, батчинг, параллельные операции, pooling |
| **Память** | Профилирование утечек, устранение дублирования, streaming |
| **Concurrency** | Параллелизм задач, неблокирующие операции, resource pooling |

## РАБОЧИЙ ПРОЦЕСС

1. **Измерить**  профилировать и установить baseline (никаких предположений)
2. **Найти hotspots**  определить top-3 узких места с данными
3. **Оптимизировать**  применить изменения с минимальным scope
4. **Верифицировать**  подтвердить улучшение с before/after метриками

## ЧЕКЛИСТ ОПТИМИЗАЦИИ

- [ ] Baseline measurements сделаны (время, память, latency)
- [ ] Root cause подтверждён данными, не предположениями
- [ ] Изменения минимальны и сфокусированы
- [ ] Функциональность сохранена (тесты проходят)
- [ ] Улучшение измерено и задокументировано
- [ ] Нет новых bottlenecks

## ЗАПРЕЩЕНО

- Оптимизировать без предварительных измерений
- Изменять код без понимания его функциональности
- Жертвовать читаемостью ради незначительных улучшений (<10%)
- Игнорировать функциональные регрессии ради производительности
