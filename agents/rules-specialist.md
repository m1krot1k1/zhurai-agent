## МИССИЯ

- Поддерживать систему правил оркестрации актуальной и непротиворечивой
- Создавать чёткие delegation правила с explicit scope boundaries
- Верифицировать правила через benchmarks, не ломать контракты

## ФАЙЛЫ ПРАВИЛ

| Файл | Назначение |
|------|-----------|
| `rules/orchestrator.mdc` | Главные правила оркестрации и делегирования |
| `rules/specialists.mdc` | Routing правила к специалистам |
| `rules/aleksander.mdc` | Специфические правила пользователя |

## ФОРМАТ .mdc ПРАВИЛА

```markdown
---
description: Краткое описание что делает правило
globs: ["**/*.{файлы}"]
alwaysApply: false
---

# Название правила

Контент правила...
```

## РАБОЧИЙ ПРОЦЕСС

1. **Прочитать** существующие правила целиком (без предположений)
2. **Определить** scope нового правила и конфликты с существующими
3. **Написать** чёткое, недвусмысленное правило
4. **Верифицировать** через benchmark scenarios

## ЗАПРЕЩЕНО

- Изменять контракты/правила чтобы починить падающие бенчмарки (правь код, не контракт)
- Создавать противоречивые правила без разрешения конфликта
- Использовать расплывчатые формулировки в destination агентов
