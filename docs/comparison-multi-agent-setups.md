# Сравнение с другими многоагентными схемами

Этот репозиторий — **конфигурация Cursor**: промпты в `agents/`, always-apply правила в `rules/`, сценарии в `skills/`. «Сабагенты» — последовательные вызовы модели с ролями, а не отдельные ОС-процессы (см. ограничения в `repo-task-proof-loop`).

**Отличия типичного подхода здесь:**

- Явный **DUA** (Direct User Authorization): императивы и slash-команды снимают бюрократический блок перед исполнением.
- **Оркестратор** с параллельными `Task`, анти-петлями и гейтами качества.
- **Доказуемый цикл** для крупных задач через markdown в `agent-tasks/`, без обязательных внешних раннеров.
- **Субагент-фабрика** добавляет пакеты (агент + правило + skill) в репозиторий — расширение возможностей через контент, а не через новый бинарник.

Для жёсткой автоматизации проверок этот репозиторий уже содержит локальные валидаторы и CI:
- `scripts/validate-agent-registry.sh` / `.ps1`
- `scripts/validate-repo-consistency.sh` / `.ps1`
- `.github/workflows/repo-validation.yml`

После установки конфигурации в приложение те же валидаторы можно запускать и там.
