# Сопоставление с Oh-My-ClaudeCode (концептуально)

**Дата:** 2026-03-30  
**Upstream (ориентир по смыслу, другой продукт):** [yeachan-heo/oh-my-claudecode](https://github.com/yeachan-heo/oh-my-claudecode). Краткое назначение по публичному README upstream: teams-first multi-agent orchestration для Claude Code — это **контекст для чтения**, не спецификация поведения **этого** репозитория.

Этот документ **не** описывает установку или код Oh-My-ClaudeCode (OMC) и **не** утверждает полную функциональную эквивалентность. Он фиксирует **rough concept parity**: где в **этом** репозитории лежат похожие идеи (маршрутизация, интенсивность прогона, рой, пайплайн, экономия токенов).

## Disclaimer

- OMC и данная конфигурация Cursor — **разные проекты**, разные runtime и границы возможностей.
- Никаких «съесть все задачи без ограничений» или обхода политик безопасности здесь нет: действуют `rules/aleksander.mdc`, инъекционные гейты и `rules/orchestrator.mdc`.
- Не копируйте маркетинговые формулировки сторонних проектов в ядро правил; смысл переносите в **измеримые** AC и проверки.

| **Swarm** | Режим супервизора: `CONTINUOUS_MODE: until_user_stop`, повторные волны handoff (root → orchestrator); внутри волны — параллельные `Task`, `B0` как supervisor | `agents/start.md` (STEP 0–2, Mode B2), `rules/orchestrator.mdc` §0 |
| **Pipeline** | Рекурсивный оркестратор: фазы разведки/извлечения/синтеза/внедрения/верификации по §12, строгие контракты веток | `rules/orchestrator.mdc` §12, `docs/delegation-chain.md` |
| **Ecomode** | Эвристики стоимости и маршрутизации: один узкий специалист vs оркестратор, параллельность, лимиты same-type write | `rules/specialists.mdc` (cost-aware routing), `rules/orchestrator.mdc` §3–§4 |

## Что здесь не претендует на parity OMC

- Любая интеграция с конкретным CLI/vendor вне описанного в репозитории.
- Гарантии «автопилота» без `Task` в runtime: см. `MULTI_AGENT_PIPELINE_BLOCKED` в `rules/orchestrator.mdc` §0.5.
- Отмена инвариантов безопасности (секреты, инъекции, high-privilege действия) — см. `rules/aleksander.mdc` §2.6.

## Как пользоваться таблицей

Имена в левом столбце — **условные ярлыки** для обсуждения с коллегами, перенесённые из экосистемы внешних обзоров; в коде этого репозитория они **не** зарезервированы как отдельные режимы. Для реальной настройки поведения опирайтесь на поля конверта (`ENTRY_MODE`, `PARENT_ROLE`, `ROOT_START_PROFILE`, `CONTINUOUS_MODE`) и на тексты в `rules/orchestrator.mdc` / `agents/start.md`.

**FULL_FORCE** не отменяет измеримые критерии приёмки: оркестратор по-прежнему обязан закрывать AC доказательствами (артефакты, команды, отчёты веток), а не «уверенностью модели».

## Сопоставление потоков (кратко)

| Шаг (упрощённо) | OMC (ожидаемый смысл в обзорах) | Этот репозиторий |
|-----------------|--------------------------------|------------------|
| Вход пользователя | Единый запрос к «автопилоту» | `/start` → root `start` → **`Task(orchestrator)`** (без worker-start) |
| Разбиение работы | Много агентов / очередь задач | `Task(orchestrator)` → ветки `Task(<specialist>)` |
| Непрерывность | Долгий прогон | `CONTINUOUS_MODE: until_user_stop`, см. `agents/start.md` |
| Контроль качества | Ревью / проверки | `code-reviewer`, `security-auditor`, benchmark-скрипты по контракту волны |
| Экономия | Меньше лишних вызовов | `rules/specialists.mdc`: cost-aware routing, параллель только безопасный |

Ни одна строка этой таблицы не импортирует исполняемый код из upstream; это **логическая** карта для читателя.

## Обновление документа

При смене канонических полей конверта (`ENTRY_MODE`, `PARENT_ROLE`, `ROOT_START_PROFILE`) правьте в первую очередь `rules/orchestrator.mdc` §1.1 и шаблон в `agents/start.md`, затем при необходимости синхронизируйте абзацы здесь — без копирования длинных цитат из внешних README.

## Границы ответственности

- **Корневой `start`** не читает репозиторий до handoff; интенсивность волны задаётся полями конверта, передаваемыми в `Task(orchestrator)` (см. `docs/delegation-chain.md`).
- **Оркестратор** отвечает за декомпозицию, параллельные ветки и соблюдение гейтов §19 при `FULL_FORCE_START_PROFILE`.
- **Специалисты** не подменяют политику безопасности и не обходят `OUT_OF_SCOPE` родителя.

Если термин из колонки «OMC-стиль» встречается в чате или issue, используйте этот файл как **глоссарий перевода** в локальные артефакты, а не как обещание совместимости с другим продуктом.
