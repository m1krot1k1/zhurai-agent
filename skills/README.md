# Skills

Workflow skills описывают повторяемые сценарии и паттерны, которыми пользуются агенты. Каждый skill — файл `SKILL.md` в своём каталоге. Всего **25 skills**.

## Реестр skills

|-------|-----------|-------------------|
| **agent-evals** | Измерения оценки агентов | Golden tasks, регрессионные наборы, рубрики; проверка перед merge core rules |
| **agent-manager** | Управление жизненным циклом агентов | Создание, обновление, деактивация, валидация реестра агентов |

| **agent-system-navigation** | Навигация по агентам | Decision tree для выбора агента, паттерны оркестрации |

## Связь со структурой

- Канонические пути этого репозитория: `agents/`, `rules/`, `docs/`, `skills/`.
- В целевом проекте эти каталоги обычно разворачиваются под `.cursor/`.
- Мастер-правило со ссылками на skills: `rules/specialists.mdc`.

## Когда создавать новый skill vs расширять существующий

**Создать новый skill** (`skills/<name>/SKILL.md`), если:
- Это самостоятельный повторяемый workflow со своим жизненным циклом (например, `repo-task-proof-loop`).
- Навык охватывает несколько шагов и используется в разных контекстах.
- Логика отличается настолько, что смешение с существующим skill затруднит понимание.

**Расширить существующий skill**, если:
- Это вариант или подпаттерн уже существующего workflow.
- Шаги идентичны, меняется только контекст применения.
- Правки небольшие и не нарушают логику существующих пользователей skill.

**Шаблон нового SKILL.md:**
```yaml
---
name: <kebab-case-name>
description: <одна строка: что делает skill и когда применять>
---
```
Затем разделы: `## Purpose`, `## Когда использовать`, `## Шаги`, `## Policy Reference` (если реализует политику из rules/).
