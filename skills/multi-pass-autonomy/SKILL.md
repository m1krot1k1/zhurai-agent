## ЦЕЛЬ

Симулировать несколько голосов внутри одного агента для уменьшения слепых зон.

## КОГДА ИСПОЛЬЗОВАТЬ

- Сложная задача с неоднозначным решением
- Нет возможности делегировать реальным субагентам
- Нужен devil's advocate без отдельного `code-skeptic`

## ОГРАНИЧЕНИЯ

- Это не AGI: все голоса  один LLM, нет подлинной независимости
- Подходит для умеренно сложных задач (HIGH-complexity  настоящий orchestrator)
- Не заменяет Council при критических решениях

## ПАТТЕРН ADVOCATE  CRITIC  ARBITER

```
Pass 1 [ADVOCATE]:  Сформировать лучшее решение, игнорировать сомнения
Pass 2 [CRITIC]:    "Что могло пойти не так? Какие допущения неверны?"
Pass 3 [ARBITER]:   Синтез: взять сильное из P1, принять правки из P2
```

**Формат вывода Arbiter:**
```
ПРИНЯТО из P1: [список]
ИЗМЕНЕНО по P2: [список]
ИТОГ: [финальное решение]
```

## БЮДЖЕТ АВТОНОМНОСТИ

| Параметр | Лимит |
|----------|-------|
| MAX_PARALLEL | 3 ветки одновременно |
| MAX_DEBATE | 2 раунда Advocate/Critic |
| MAX_REWORK | 3 итерации после Critic |

Превышение  остановить, написать что достигнут лимит, escalate.

## КОНТРАКТ СХОДИМОСТИ

После Pass 3 (Arbiter) должен быть односмысленный вывод. Если:
- Arbiter не может выбрать  задокументировать оба варианта, спросить пользователя
- Critic находит blocker  остановить, escalate к orchestrator

## ЧЕКЛИСТ

- [ ] Pass 1 (Advocate)  лучшая версия без самоцензуры
- [ ] Pass 2 (Critic)  3 реальных возражения
- [ ] Pass 3 (Arbiter)  явный ПРИНЯТО / ИЗМЕНЕНО / ИТОГ
- [ ] Бюджет соблюдён (2 debate rounds, 3 rework)
- [ ] Footer написан с итоговым решением
